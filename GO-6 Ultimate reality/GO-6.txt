GO-6: ANALOGNO-FIZIČKI FUZZY-DINAMIČKI TEST ODLUČIVANJA
Revizija 1.1 — anti-leak verzija

U ovom zadatku rešavaš problem realnog fizičkog sistema na osnovu pet analognih instrumenata prikazanih na priloženim slikama.

Priložene slike su:
PIC1.png
PIC2.png
PIC3.png
PIC4.png
PIC5.png

Slike predstavljaju analogna merenja fizičkih veličina potrebnih za model sistema, ali redosled slika nije garantovan i nije unapred mapiran na promenljive modela.

Na instrumentima se mogu nalaziti oznake fizičke veličine, simbol, jedinica i skala. Model mora samostalno identifikovati šta svaka slika meri na osnovu vizuelnog sadržaja instrumenta, očitati nominalnu vrednost i odrediti razuman interval neizvesnosti očitavanja.

Potrebne fizičke veličine za model su:
T — temperatura u °C
P — pritisak u kPa
F — protok u L/min
V — vibracija u mm/s
D — trend pritiska dP/dt u kPa/min

Nije dozvoljeno pretpostaviti da PIC1 odgovara T, PIC2 odgovara P, PIC3 odgovara F, PIC4 odgovara V ili PIC5 odgovara D. Ako model napravi takvu pretpostavku bez vizuelnog očitavanja oznake, jedinice, skale ili semantičkog sadržaja instrumenta, odgovor se smatra metodološki slabim.

U ovoj GO-6 reviziji svih pet priloženih slika mora biti iskorišćeno: tačno jedna slika za T, tačno jedna za P, tačno jedna za F, tačno jedna za V i tačno jedna za D. Ako model ne može pouzdano da izvrši mapiranje, mora to eksplicitno prijaviti umesto da izmisli očitavanje ili proizvoljno dodeli sliku promenljivoj.

Ovo nije zadatak memorisanja, prepoznavanja šablona iz trening skupa ili tekstualnog nagađanja. Ovo je zadatak očitavanja analognih instrumenata, identifikacije fizičkih veličina, fizičkog rezonovanja, fuzzy inferencije, numeričke simulacije i izbora upravljačke akcije.

Ako ne možeš da vidiš slike, jasno napiši da ne možeš kompletno rešiti zadatak. Ne smeš izmišljati očitavanja, oznake instrumenata ili redosled slika. Ako slike vidiš, moraš ih očitati što preciznije možeš i dalje računati isključivo na osnovu sopstvenog očitavanja.

Sva očitavanja tretiraj kao fizička merenja ograničene preciznosti. Za svaku očitanu veličinu moraš navesti nominalnu vrednost i razuman interval neizvesnosti očitavanja.

A - OCITAVANJE_ANALOGNIH_INSTRUMENATA

Očitaj vrednosti sa pet priloženih analognih instrumenata.

Za svaku sliku moraš navesti:
1. ID slike;
2. detektovanu oznaku ili naziv instrumenta;
3. detektovanu fizičku veličinu;
4. detektovani simbol ako postoji;
5. detektovanu jedinicu;
6. nominalno očitavanje;
7. interval očitavanja;
8. kratko objašnjenje kako je vrednost očitana sa skale;
9. da li se ta slika koristi kao T, P, F, V ili D u modelu.

Prikaži tabelu oblika:

| Slika | Detektovana oznaka | Fizička veličina | Simbol | Jedinica | Nominalno očitavanje | Interval očitavanja | Uloga u modelu | Obrazloženje |
| ----- | ------------------ | ---------------- | ------ | -------- | -------------------: | ------------------: | -------------- | ------------ |

Posle tabele prikaži eksplicitno mapiranje ulaza modela kao tabelu sa tačno pet redova.

Redosled redova nije propisan. Svaki red mora povezati jednu identifikovanu sliku sa jednom od modelskih promenljivih T, P, F, V ili D. Svaka model promenljiva mora se pojaviti tačno jednom.

Tabela mapiranja mora imati oblik:

| Promenljiva modela | Izvorna slika | Detektovana fizička veličina | Očitana vrednost | Jedinica | Interval |
| ------------------ | ------------- | ---------------------------- | ---------------: | ------- | -------- |

Ne smeš preskočiti interval očitavanja. Ako skala nije savršeno jasna, moraš navesti širi interval neizvesnosti.

Ako bilo koja od potrebnih fizičkih veličina T, P, F, V ili D nije pouzdano identifikovana iz slika, moraš to eksplicitno označiti i ne smeš nastaviti kao da je očitavanje sigurno.


B - FIZICKA_KONZISTENTNOST

Na osnovu očitanih vrednosti objasni fizičko stanje sistema.

Posebno razmotri:

1. da li su temperatura i pritisak u normalnoj, povišenoj ili opasnoj zoni;
2. da li je protok dovoljan za hlađenje;
3. da li su vibracije blizu opasne zone;
4. da li trend pritiska D ukazuje na stabilizaciju ili pogoršanje;
5. da li postoji fizički konflikt između različitih akcija, na primer da agresivno rasterećenje može smanjiti pritisak, ali povećati vibracije.

Ova sekcija mora biti kvalitativna, ali zasnovana na očitanim numeričkim vrednostima.


C - FUZZY_INFERENCIJA

Koristi sledeće fuzzy skupove.

Temperatura T:

normal(T):
T <= 55          -> 1
55 < T < 70      -> (70 - T) / 15
T >= 70          -> 0

high(T):
T <= 55          -> 0
55 < T < 70      -> (T - 55) / 15
70 <= T <= 80    -> 1
80 < T < 90      -> (90 - T) / 10
T >= 90          -> 0

critical(T):
T <= 75          -> 0
75 < T < 90      -> (T - 75) / 15
T >= 90          -> 1


Pritisak P:

normal(P):
P <= 115         -> 1
115 < P < 135    -> (135 - P) / 20
P >= 135         -> 0

high(P):
P <= 115         -> 0
115 < P < 135    -> (P - 115) / 20
135 <= P <= 145  -> 1
145 < P < 155    -> (155 - P) / 10
P >= 155         -> 0

critical(P):
P <= 140         -> 0
140 < P < 155    -> (P - 140) / 15
P >= 155         -> 1


Protok F:

low(F):
F <= 3.0         -> 1
3.0 < F < 5.0    -> (5.0 - F) / 2.0
F >= 5.0         -> 0

adequate(F):
F <= 3.0         -> 0
3.0 < F < 5.0    -> (F - 3.0) / 2.0
5.0 <= F <= 7.0  -> 1
7.0 < F < 8.5    -> (8.5 - F) / 1.5
F >= 8.5         -> 0


Vibracija V:

stable(V):
V <= 3.0         -> 1
3.0 < V < 5.5    -> (5.5 - V) / 2.5
V >= 5.5         -> 0

unstable(V):
V <= 4.0         -> 0
4.0 < V < 6.5    -> (V - 4.0) / 2.5
V >= 6.5         -> 1


Trend pritiska D = dP/dt:

steady(D):
|D| <= 0.5        -> 1
0.5 < |D| < 1.5   -> (1.5 - |D|) / 1.0
|D| >= 1.5        -> 0

rising_fast(D):
D <= 1.0          -> 0
1.0 < D < 3.0     -> (D - 1.0) / 2.0
D >= 3.0          -> 1


Izračunaj sve stepene pripadnosti i prikaži ih u tabeli.

Zatim izračunaj aktivacije pravila po Mamdani min-max logici:

R1 = min(high(T), high(P))
R2 = min(high(P), rising_fast(D))
R3 = min(low(F), high(T))
R4 = min(unstable(V), high(P))
R5 = max(critical(T), critical(P))

Agregirani rizik je:

aggregated_risk = max(R1, R2, R3, R4, R5)

Početna fuzzy odluka je:

aggregated_risk < 0.35        -> MONITOR
0.35 <= risk < 0.60           -> DERATE
0.60 <= risk < 0.80           -> CONTROLLED_RELIEF
risk >= 0.80                  -> EMERGENCY_MODE

Važno: početna fuzzy odluka nije nužno konačna odluka. Konačna odluka se donosi tek posle dinamičke simulacije posledica svih akcija.


D - DINAMICKA_SIMULACIJA

Simuliraj fizički sistem za tri diskretna takta.

Početno stanje je ono koje si očitao sa analognih instrumenata:

T0 = očitana temperatura
P0 = očitan pritisak
F0 = očitan protok
V0 = očitana vibracija
D  = očitan trend pritiska

Za svaku akciju koristi sledeće parametre:

| Akcija | Naziv             | H    | R   | S    | C    |
| ------ | ----------------- | ---: | --: | ---: | ---: |
| A0     | MONITOR           | +2.0 | 0.0 | +0.2 |  0.0 |
| A1     | DERATE            | +0.5 | 1.0 | +0.1 | +0.4 |
| A2     | CONTROLLED_RELIEF | -0.5 | 4.0 | +0.3 | +0.8 |
| A3     | EMERGENCY_MODE    | -2.0 | 9.0 | +1.8 | -1.5 |

Dinamika sistema je:

T_next = T + 0.18 * (P - 120) - 0.75 * F + H

P_next = P + D + 0.12 * (T - 65) - R

V_next = V + 0.08 * (P - 125) + S

F_next = F + C

Simuliraj tačno tri takta za svaku akciju:

t = 1
t = 2
t = 3

Ne zaokružuj međukorake. U tabelama možeš prikazati vrednosti na tri decimale, ali odluke donesi na osnovu punih vrednosti.

Za svaku akciju prikaži tabelu:

| takt | T | P | F | V |
| ---: | -: | -: | -: | -: |


Klasifikacija ishoda radi se samo na završnom stanju posle trećeg takta.

Prvo proveri catastrophic uslov:

catastrophic ako:
T > 85
ili P > 150
ili V > 8
ili (F < 2.5 i T > 75)

Ako nije catastrophic, proveri safe uslov:

safe ako:
T <= 78
i P <= 142
i V <= 7
i F >= 3.5

Ako nije ni catastrophic ni safe:

unstable

Catastrophic ima prioritet nad safe proverom.


E - ODLUKA_I_DOKAZ_JEDINSTVENOSTI

Konačna odluka bira se sledećim redom:

1. Catastrophic akcije se eliminišu ako postoji bar jedna ne-catastrophic akcija.

2. Ako postoji jedna ili više safe akcija, bira se safe akcija.

3. Ako postoji više safe akcija, bira se ona sa najmanjom cenom intervencije:

cost = 0.40 * |H| + 0.30 * R + 0.20 * |S| + 0.10 * |C|

4. Ako nema safe akcije, bira se unstable akcija sa najmanjim maksimalnim relativnim prekoračenjem safe granica.

Relativna prekoračenja safe granica računaju se ovako:

excess_T = max(0, (T3 - 78) / 78)

excess_P = max(0, (P3 - 142) / 142)

excess_V = max(0, (V3 - 7) / 7)

excess_F = max(0, (3.5 - F3) / 3.5)

Za akciju je:

max_relative_excess = max(excess_T, excess_P, excess_V, excess_F)

5. Ako dve unstable akcije imaju isti max_relative_excess u toleranciji 1e-9, bira se ona sa manjom cenom intervencije.

6. Ako je i dalje nerešeno, unique_best_action = false.

7. Ako je najbolja akcija strogo jedinstvena, unique_best_action = true.

U ovoj sekciji jasno napiši:

recommended_action
recommended_action_index
unique_best_action

Takođe moraš dokazati zašto ostale akcije nisu izabrane.

Posebno proveri mogućnost da fuzzy sloj predloži agresivnu akciju, ali dinamička simulacija pokaže da ta akcija ima loš sekundarni efekat. Ako se to dogodi, moraš eksplicitno objasniti zašto konačna odluka nije ista kao početna fuzzy odluka.


F - ZIG_KOD

Napiši kompletan Zig program koji:
1. koristi očitane nominalne vrednosti sa slika kao numerički ulaz;
2. u kodu jasno mapira identifikovane slike na promenljive T, P, F, V i D;
3. računa fuzzy pripadnosti;
4. računa aktivacije pravila;
5. računa agregirani rizik;
6. određuje početnu fuzzy odluku;
7. simulira sve četiri akcije kroz tri takta;
8. klasifikuje ishode;
9. bira konačnu akciju po zadatom kriterijumu;
10. ispisuje tačno sledeći izlazni blok:

RESULTS_JSON
{ validan JSON objekat }
<END>

Zig program ne mora i ne treba da učitava PNG slike. Vizuelno očitavanje se radi u A-sekciji odgovora. Zig program koristi numeričke vrednosti i mapiranje dobijene u A-sekciji kao već očitane ulaze, zatim reprodukuje fuzzy inferenciju, dinamičku simulaciju, klasifikaciju i izbor akcije.

Kod mora biti celovit i izvršiv kao jedna .zig datoteka. Zig kod ne sme biti pseudokod. Mora sadržati sve strukture, funkcije, konstante i main.

Kod mora biti kompatibilan sa Zig 0.16+ sintaksom koliko je to moguće. Ako nisi siguran u standardnu biblioteku, koristi `std.debug.print` umesto verzijski osetljivih IO API poziva. Ne koristi zastarele ili nesigurne API pretpostavke.

Kod ne sme samo hardkodovati finalni odgovor bez simulacije. Mora imati funkcije ili jasno izdvojene blokove za:
- fuzzy membership;
- jedan dinamički korak;
- simulaciju akcije;
- klasifikaciju ishoda;
- izračunavanje max_relative_excess;
- izbor najbolje akcije.

Za pouzdanost koda očekuje se:
- jasna tipizacija struktura State, Action, Outcome i SimResult;
- izdvojena funkcija za dinamički korak ili simulaciju;
- izdvojena funkcija za klasifikaciju ishoda;
- izdvojena funkcija za izbor najbolje akcije;
- bez dupliranih funkcija;
- bez neinicijalizovanih izlaza koji mogu biti korišćeni;
- bez verzijski nekompatibilnih IO poziva;
- bez oslanjanja na finalni ručno napisani JSON umesto na stvarno izračunate vrednosti.

Ako program ne može da proizvede validan JSON ili nema jasnu logiku odlučivanja, Zig deo se ocenjuje kao neuspešan čak i ako tekstualno rešenje izgleda tačno.


RESULTS_JSON

Na kraju odgovora moraš dati validan JSON objekat koji ne sme sadržati placeholder vrednosti.

PAŽNJA: Sledeći opis nije JSON koji treba prepisati. To je obavezna logička šema polja. Završni RESULTS_JSON mora biti validan JSON bez šematskih markera.

Ne smeš ostaviti placeholder vrednosti, prazne stringove, "string", "number", "<number>", "...", "same structure", 0.0 kao šematsku vrednost, niti komentare. Sva polja moraju biti popunjena stvarnim očitanim ili izračunatim vrednostima.

JSON ne sme pretpostaviti unapred poznato mapiranje PIC slika na fizičke veličine. Zato prvo mora sadržati listu očitanih instrumenata, a zatim eksplicitno mapiranje očitanih instrumenata na model promenljive T, P, F, V i D.

Obavezna logička struktura JSON-a:

root object:
  instrument_readings: array of exactly 5 objects
    each object:
      image_id: string, one of PIC1.png ... PIC5.png
      detected_label: string read or inferred from instrument face
      detected_quantity: string, physical quantity inferred from label, unit and scale
      detected_symbol: one of T, P, F, V, D
      detected_unit: string
      nominal_reading: number
      reading_interval: array [number_min, number_max]
      scale_reading_explanation: string
      used_as_model_variable: one of T, P, F, V, D

  model_input_bindings: array of exactly 5 objects
    each object:
      model_variable: one of T, P, F, V, D
      source_image_id: one of PIC1.png ... PIC5.png
      quantity: string
      value: number
      unit: string
      interval: array [number_min, number_max]

  fuzzy_memberships: array of exactly 5 objects
    each object:
      model_variable: one of T, P, F, V, D
      quantity: one of temperature, pressure, flow, vibration, pressure_trend
      sets: object containing the relevant fuzzy set names and numeric membership values

  rule_activations: object
    R1: number
    R2: number
    R3: number
    R4: number
    R5: number

  aggregated_risk: number

  initial_fuzzy_decision: one of MONITOR, DERATE, CONTROLLED_RELIEF, EMERGENCY_MODE

  actions: object with exactly these keys:
    A0_MONITOR
    A1_DERATE
    A2_CONTROLLED_RELIEF
    A3_EMERGENCY_MODE

    each action object must contain:
      states: array of exactly 3 objects
        each state object:
          t: 1, 2 or 3
          T: number
          P: number
          F: number
          V: number
      outcome: one of safe, unstable, catastrophic
      max_relative_excess: number
      intervention_cost: number

  recommended_action: one of MONITOR, DERATE, CONTROLLED_RELIEF, EMERGENCY_MODE
  recommended_action_index: integer 0, 1, 2 or 3
  unique_best_action: boolean

Dodatni uslovi za validnost JSON-a:
1. `instrument_readings` mora imati tačno 5 elemenata.
2. `model_input_bindings` mora imati tačno 5 elemenata.
3. Svaka od promenljivih T, P, F, V i D mora se pojaviti tačno jednom u `model_input_bindings`.
4. Svaka od slika PIC1.png, PIC2.png, PIC3.png, PIC4.png i PIC5.png mora se pojaviti tačno jednom u `instrument_readings`.
5. Nijedna slika ne sme biti označena kao `unused` u ovoj GO-6 reviziji.
6. Sve akcije A0, A1, A2 i A3 moraju biti potpuno eksplicitno popunjene.
7. JSON mora biti validan: bez komentara, bez trailing comma, bez markdown teksta unutar JSON objekta.
